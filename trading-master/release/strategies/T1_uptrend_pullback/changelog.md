# WARREN-T1 上涨趋势波段接力系统 - 版本变更记录

## v1.1 (2026-04-09) - 策略库迁移与完整参数化

**状态**：✅ 正式发布 / 策略库归档版本

### 变更概览
本次版本主要完成策略从概念到可执行系统文档的完整归档，并建立了参数化配置文件，为未来回测和自动化执行奠定基础。

### 新增功能
1. **参数配置文件 (`params.json`)**：
   - 将所有核心参数抽取为结构化JSON配置
   - 包含市场过滤器、入场、加仓、离场、风险等完整模块
   - 支持未来通过修改配置文件快速调整策略

2. **策略库标准化结构**：
   - 创建独立的策略目录 `T1_uptrend_pullback`
   - 建立标准文件组织：`README.md`, `trading_manual.md`, `system_technical.md`, `params.json`, `changelog.md`, `tdx_formula.json`
   - 分离 `backtest/` 和 `signals/` 子目录用于未来数据存储

3. **通达信公式完善**：
   - 将原公式优化为JSON格式，增加详细说明和备注
   - 明确公式使用场景和限制条件

4. **版本记录系统**：
   - 建立规范的变更记录模板

### 策略逻辑完善
1. **明确状态机重置规则**：
   - 每次完全清仓后视为全新独立循环
   - 必须重新进行环境过滤
   - 每次新交易必须动用基础风险1R（2500元）

2. **补充主动离场策略**：
   - 明确达到2R盈亏比时平半仓
   - 剩余仓位转入移动止损托管
   - 解决了原系统仅依赖被动止损的问题

3. **双引擎加仓逻辑细化**：
   - 明确逼空不回踩与标准缩量回踩两种场景
   - 规定两种场景下止损上移的具体位置
   - 保持金字塔递减原则（加仓风险为1250元）

4. **风险额度管理强化**：
   - 单笔基础风险额度（1R）锁定为2500元人民币
   - 仓位完全由风险计算动态确定
   - 确保任何单笔损失不超过预设额度

### 技术架构改进
1. **模块化设计**：
   - 四阶段工作流：环境过滤 → 双轨入场 → 双引擎加仓 → 混合离场
   - 各模块职责清晰，可独立测试和优化

2. **量化规则明确**：
   - 所有条件均有具体数值定义
   - 消除主观判断空间
   - 基于A股统计特征设定参数

3. **文档完整性**：
   - 实战手册：面向交易员的执行指南
   - 技术文档：系统设计原理和实现细节
   - 公式文件：自动化筛选工具

### 已知限制
1. 尚未进行大规模历史回测验证
2. 盘中量能预估依赖人工判断
3. 市场环境过滤器在极端行情下可能失效
4. 未考虑大盘系统性风险叠加

## v1.0 (2026-04-08) - 系统原型

**状态**：⏸️ 已归档 / 被v1.1取代

### 初始功能
1. **核心概念形成**：上涨趋势中的波段接力
2. **环境过滤器原型**：趋势+形态+回撤+量能四重过滤
3. **双轨入场机制**：激进突袭与稳健确认
4. **基础风险控制**：单笔损失不超过2500元

### 主要缺陷
1. 缺少主动离场策略，仅依赖移动止损
2. 状态机重置规则不明确
3. 参数未完全量化，存在主观判断空间
4. 文档分散，缺乏统一管理

### 变更原因
v1.0作为概念验证，通过沙盘推演发现多个逻辑漏洞和模糊点，需要全面系统化才能实际使用。

## 版本管理规则

### 版本号约定
- **主版本号**：重大架构调整或哲学改变
- **次版本号**：功能增加或重要逻辑完善
- **修订号**：bug修复或参数微调

### 发布流程
1. 沙盘推演验证新版本逻辑
2. 更新所有相关文档
3. 修改参数配置文件
4. 记录变更日志
5. 进行模拟盘测试（如条件允许）

### 回滚机制
- 每个版本的完整文件集应独立备份
- 参数配置变更前必须保存旧版本
- 重大变更应先在分支目录测试
